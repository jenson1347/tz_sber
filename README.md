# tz_sber
# 📊 Hybrid Intent Classification Pipeline (Cascading & Blending)

Репозиторий содержит production-ready пайплайн гибридной классификации интентов пользователей. Архитектура построена по принципу **каскадной фильтрации (Cascading Classification)** и **динамического взвешивания моделей (Probability Blending)** с возможностью эскалации сложных запросов в LLM. Это позволяет минимизировать вычислительные затраты и максимизировать точность ответов.

## 🗺️ 1. Пайплайн проекта (Схема работы каскада)

Ниже представлена логическая схема прохождения входящего текстового запроса через три уровня фильтрации:

```text
       [ Входящий запрос (Текст клиента) ]
                       │
                       ▼
         ┌───────────────────────────┐
         │     TF-IDF Экстракция     │
         └─────────────┬─────────────┘
                       │
                       ▼
         ┌───────────────────────────┐
         │     CatBoost Classifier   │ ───► Расчет Margin Confidence:
         └─────────────┬─────────────┘       (P_top1 - P_top2)
                       │
                       ├───────────────────────────────┐
                       │ (Если Margin >= cb_threshold) │ (Если Margin < cb_threshold)
                       ▼                               ▼
            ┌─────────────────────┐        ┌───────────────────────┐
            │ ⚡ Быстрый ответ     │        │  Токенизация & Прогон │
            │    (CatBoost)       │        │    через RuBERT       │
            └─────────────────────┘        └───────────┬───────────┘
                                                       │
                                                       ▼
                                           ┌───────────────────────┐
                                           │  Динамический блендинг│
                                           │   вероятностей по     │
                                           │   весам Softmax       │
                                           └───────────┬───────────┘
                                                       │
                       ┌───────────────────────────────┴──────────────────────────────┐
                       │ (Если Max_Prob >= llm_threshold)                             │ (Если Max_Prob < llm_threshold)
                       ▼                                                              ▼
           ┌──────────────────────┐                                       ┌───────────────────────┐
           │ 🤖 Ответ Ансамбля    │                                       │ 🔮 Генерация ответа   │
           │ (CatBoost + RuBERT)  │                                       │      через LLM        │
           └──────────────────────┘                                       └───────────────────────┘

##2. Описание проекта
Проект решает классическую задачу клиентского сервиса — автоматическое определение интента (намерения) пользователя по его текстовому сообщению в чате.

Основная проблема классических подходов в NLP — компромисс между скоростью работы и точностью. Тяжелые трансформеры (BERT, LLM) дают высокое качество, но требуют GPU и долго думают. Легкие классификаторы (CatBoost) работают мгновенно на CPU, но ошибаются на сложных/зашумленных запросах. Данный проект решает эту проблему за счет гибридного каскада.


3. Принятые архитектурные решения
В ходе разработки пайплайна были приняты следующие ключевые решения: Метрика фильтрации Margin Confidence вместо Max Probability. 
Для отсечения простых запросов на первом этапе CatBoost оценивает не максимальную вероятность класса, а разница между топ-1 и топ-2 предсказаниями: Margin = P_top1 - P_top2. Это защищает систему от ситуаций, когда модель выдает высокую, но неуверенную вероятность в условиях сильного шума.
Динамический блендинг на основе статических весов Softmax:
    Для совместной работы CatBoost и BERT вероятности моделей складываются с учетом их глобальной силы на тестовом датасете. Статические веса рассчитываются через Softmax от исторического Accuracy моделей
Автоматический подбор порогов (optimize_thresholds.py):
    Пороги cb_threshold и llm_threshold не хардкодятся, а рассчитываются на тестовой выборке. Целевая функция оптимизации — достижение общей точности системы при минимально возможном перенаправлении трафика в дорогую LLM API.
4. Описание моделей и их параметры
УРОВЕНЬ 1: TF-IDF Векторизатор + CatBoost
    Используется как базовый быстрый классификатор текста. Применяется символьный и словесный n-gram анализ текста.TF-IDF Параметры:ngram_range: (1, 2) (учитывает как отдельные слова, так и биграммы)
        max_features: 5000 (ограничение размера словаря для предотвращения переобучения)
        analyzer: 'word'
        CatBoostClassifier Параметры:
        loss_function: 'MultiClass'
        iterations: 1000 (количество деревьев)
        learning_rate: 0.05
        depth: 6 (оптимальная глубина для текстовых эмбеддингов)
        task_type: 'CPU' (для гарантированной переносимости в Docker)
УРОВЕНЬ 2: RuBERT-tiny2 Легковесный и быстрый русскоязычный трансформер (всего 29 млн параметров), предназначенный для извлечения глубокого контекста из «грязных» и нестандартных запросов, которые пропустил CatBoost.Параметры модели:
    base_model: 'cointegrated/rubert-tiny2'
    max_length: 64 (оптимальная длина для коротких банковских интентов)
    batch_size: 32
    optimizer: AdamW с learning_rate = 2e-5
    epochs: 5 (с сохранением чекпоинта по минимальному val_loss)
УРОВЕНЬ 3: Large Language Model (LLM)Используется в качестве «высшего арбитража» для сложных, длинных, нетипичных или маргинальных запросов, по которым ансамбль моделей выдал суммарную уверенность ниже llm_threshold.Конфигурация запроса:temperature: 0.0 (строго детерминированный ответ для классификации)max_tokens: 20 (ограничение на длину ответа для экономии токенов)Системный промт (System Prompt):PlaintextТы — интеллектуальный классификатор банковских обращений Сбера.

## 🛠️ 5. Автоматический запуск (Оркестрация)

Поскольку тяжелые файлы моделей не хранятся в Git, репозиторий снабжен скриптом автоматизации `run_pipeline.sh`. При первом запуске Docker-контейнера скрипт самостоятельно проверяет наличие весов, запускает обучение и калибровку порогов, после чего поднимает инференс:

```bash
# 1. Сборка Docker-образа под CPU инференс
docker build -t sber_intent_cascade:latest .

# 2. Запуск полного цикла (обучение -> расчет весов -> тест предиктора)
docker run -it --rm sber_intent_cascade:latest
